# News Aggregator Skill

聚合多源实时热点，输出通用中文简报。

## 0) 交互规则（强制）

- 默认只提供 Claude Code 对话式用法，不展示脚本参数。
- 仅当用户明确要求命令行细节时，才给 CLI 指令。

## 1) 能力范围

- 多源聚合：科技、开源、财经、社交热榜、扩展 RSS。
- 质量增强：去重、规则排序、可选二次重排。
- 深度阅读：可抓取正文用于更完整解读。
- 报告输出：生成结构化中文简报并可保存到 `reports/`。

## 2) 触发方式

- 菜单触发：`news-aggregator-skill 如意如意`
- 自然语言触发（示例）：
  - `帮我看今天的 AI 新闻`
  - `全网扫描今天科技热点`
  - `看下国内科技和市场的重点新闻`

## 3) 菜单行为

当用户请求菜单/帮助时：

1. 读取 `templates.md`。
2. 原样展示菜单。
3. 引导用户回复序号或直接说自然语言需求。

## 4) 执行策略（简化版）

- 根据用户目标自动选择来源与覆盖范围。
- 先做广覆盖抓取，再做去重和排序。
- 当用户明确要求“最重要/精选”时，开启二次重排。
- 当时间窗口结果过少时，补充高价值条目并明确标注。

## 4.5) 二次重排（可用）

- 采用两阶段：`规则初排 -> LLM 二次重排`。
- 重排维度：影响力、时效性、可信度、相关性。
- 若重排失败，自动降级到规则排序，保证稳定输出。

## 5) 输出格式规范

- 语言：简体中文。
- 风格：简洁、可读、可执行。
- 推荐结构：`Global Headlines`、`Tech & AI`、`Finance / Social`。

每条新闻默认包含：

- 标题（Markdown 链接）
- 来源 + 时间 + 热度/评分
- 简要摘要（突出“发生了什么 + 为什么重要”）
- 主题标签（自动生成 3-5 个，不固定词表）
- 可选：一行入选理由（当用户要求“精选”时）

主题生成原则：

- 自动从新闻内容提取实体与关键词。
- 优先保留模型名、公司名、产品名、技术名。
- 不强制白名单，不绑定个人偏好。

## 6) 重要例外

- GitHub Trending 这类榜单源按抓取结果直接呈现，不做补量。
- 单源失败不影响整批输出，继续返回可用结果。
